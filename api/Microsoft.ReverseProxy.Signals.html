Namespace Microsoft.ReverseProxy.Signals

Classes

Signal<T>

Represents a signal that holds a value of type T and serves as a notification mechanism when the value changes. This is especially useful with immutable type T so that once a reference is obtained from this class using Value or GetSnapshot(), the value can be used without worries of concurrent changes.

SignalContext

Represents the context in which a signal was created. Signal operations can only be performed among signals in the same context to ensure thread safety.

SignalExtensions

Extension methods for IReadableSignal<T>.

SignalFactory

Class used to create instances of Signal<T> within the same SignalContext.

Unit

Dummy value which can be used as a type parameter with Signal<T> to create a value-less signal, useful to propagate notification events.

Interfaces

IReadableSignal<T>

Represents a signal whose value can be read at any time, and which can interoperate with any other signals belonging to the same SignalContext.

ISignal

All signals implement this interface, which includes a property Context indicating to which context the signal belongs. Signals from different contexts cannot be mixed due to thread safety concerns.

ISignalSnapshot<T>

Snapshot of a Signal<T> allowing to subscribe to state changes.

IWritableSignal<T>

Represents a signal whose value can be set at any time, and which can interoperate with any other signals belonging to the same SignalContext.
